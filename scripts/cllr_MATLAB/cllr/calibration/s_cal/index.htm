S-Cal

 

This is a discriminatively trainable calibration mapping, to convert detection scores to detection log-likelihood-ratios.

 

This form of calibration is suitable for scores that are already ‘close’ in some sense to log-likelihood-ratios. An example would be a T-normed GMM score.  In most cases a simple linear calibration mapping (shifting and scaling) will convert this kind of score to an acceptable log-likelihood-ratio. But for good measure, we include soft saturation of the log-likelihood-ratio. This is a good idea if the quality of the log-likelihood-ratio will be evaluated via Cllr. Where does this particular mapping come from? Practically it seems to be a good idea: the saturation avoids high Cllr penalties. But here is where the idea comes from.  

Training S-cal

S-cal can be discriminatively trained on a set of development scores. This means the S-cal parameters  are optimized so that the development scores mapped to log-likelihood-ratios via S-cal, minimize the objective Cllr.

 

The whole procedure can be summarized as follows:

-       Select a set of supervised calibration trials (target trials and non-target trials) to be used for this purpose. The selection of these trials is very important:

o      The trials must be as similar as possible to the trials for which this calibration will be employed in future.

o      The trials must be sufficiently numerous. (You will probably find that as you use fewer trials for this purpose, the magnitudes of  a and b become smaller.)

o      These trials must (ideally) not have been used for other system-wide training, such as the training of fusion parameters to combine different detection sub-systems. The higher the capacity (roughly the number of parameters) of such system-wide training, the more is the danger that the scores obtained from these trials will be unsuitable for calibration purposes.

-       Run the system under calibration on these trials, to obtain sets of target and non-target scores.

-       Use these scores as input to the MATLAB function train_s_cal.m. The output is the parameter set . These parameters can then be used to map new scores:

 

Calibrating new scores

To map new scores to log-likelihood-ratios, simply call s_cal.m with the trained parameters  and the scores to be mapped.

How does it work?

The function train_s_cal.m discriminatively optimizes the parameters to minimize the objective Cllr on the given training scores. For simplicity, we used a numerical optimization routine (see below) which does not require the use of derivatives of the objective function.

 

Note that we found it necessary to regularize the objective function by adding a penalty . If is zero, the minimization objective may be ill-conditioned. For details, see the code. 

 

Software

-       You will need several of the functions in FoCal.zip.

-       You will also need the numerical optimization functional fmins.m, which is distributed with MATLAB. (We tested it in MATLAB version 5.2). If you do not have fmins.m available, you can implement it yourself by using, for example, the fairly simple Nelder-Mead downhill simplex method, as described in the Numerical Recipes series of books. (Since S-cal is differentiable with respect to its parameters, other more sophisticated gradient descent methods may also be used.)
